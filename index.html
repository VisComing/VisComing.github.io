调研并阐述SRv6的原理细节

SRv6是如何与IPv6结合的

SDN技术将网络层数据平面和控制平面分离。在控制平面上，使用SR对网络进行控制；在数据平面上，使用IPv6协议进行数据的转发，从而实现了SR与IPv6的结合。

IPv6报文头支持扩展，因此可通过扩展报文头的方式在报文头中增加SRv6的控制信息，从而实现了SR与IPv6的结合。

SRv6相关的数据结构

SR Header

为了在IPv6报文中实现SRv6转发，引入了一个SRv6扩展头（Routing Type为4），叫Segment Routing Header（SRH），用于进行Segment的编程组合形成SRv6路径。

下图为SRv6的报文格式。绿色部分-IPv6报文头；黄色部分-SRH；蓝色部分-报文内容。

IPv6 Next Header字段取值为43，表示后接的是IPv6路由扩展头。Routing Type = 4，表明这是SRH的路由扩展头，这个扩展头里字段解释如下：

Segment List中包含n个Segment，其中包含网络节点的信息，从而指定了一条SR路径。Segments Left（SL）的初始值为n，在转发过程中，每到达一个网络节点，Segments Left的值减1。

Segment ID

上图中的Segment List由多个Segment组成。

标识SRv6 Segment的ID被称为SRv6 SID。SRv6 SID是一个128bit的值，由Locator、Function和Arguments三部分组成。

Locator（位置标识）：网络中分配给一个网络节点的标识，可以用于路由和转发数据包。Locator有两个重要的属性，可路由和聚合。在SRv6 SID中Locator是一个可变长的部分，用于适配不同规模的网络。

Function（功能）：设备分配给本地转发指令的一个ID值，该值可用于表达需要设备执行的转发动作，相当于计算机指令的操作码。在SRv6网络编程中，不同的转发行为由不同的功能ID来表达。一定程度上功能ID和MPLS标签类似，用于标识VPN转发实例等。

Args（变量）：转发指令在执行的时候所需要的参数，这些参数可能包含流，服务或任何其他相关的可变信息。

SRv6如何对流量进行控制

指令集

每个SID与一个指令绑定，告知节点在处理SID时需要执行的动作（转发、封装、解封装）

SRv6转发过程

源节点：为IP报文封装IPv6报文头和SRH，然后进行转发。其中，执行H指令动作，将SRv6路径信息封装在SRH中，指定路径中各节点的SID，并将下一节点的SID拷贝到IPv6报文头的目的地址字段。

EndPoint节点：收到报文后，根据IPv6报文头中的目的地址查找本地SID表（每个SRv6节点维护一张本地SID表），若命中，则执行相应的End指令动作；然后令SL的值减1，并将SL所指示的SID拷贝到IPv6报文头的目的地址字段，并向下一跳转发。

解释步骤4中 ping6 操作的流量过程

ping请求路径

执行ping6后，首先会执行ND协议来获得目的IP地址的MAC地址，如下图所示

获取目的MAC地址后，就会构造ICMPv6报文了，下图为主机a构造的原始的ICMPv6包，仅有一个IPv6首部。

经过R1路由器时，由于R1路由器配置了SRv6，所以会将该普通的IPv6报文封装成SRv6报文，添加SRv6首部

可以看到加了IPv6首部，其中首部类型是4(Segment Routing)，里面有两个Segment[fc00:4::bb, fc00:3::bb]，注意到Segments Left为1，所以该首部里面的目的地址就是fc00:3::bb

由于R2路由器没有配置SRv6，所以R2会根据IPv6报头中的目的地址进行转发，不会对IPv6的报头做出修改。

在R3，R3路由器根据Segment执行End操作，将Segment Left减1，并根据Segment列表更新IPv6的目的地址，将数据包转发至下一跳R4。下图为从R3从eth3发出的包，同时也是R4从eht2接收到的包

R4上收到R3发来的数据包，由于Segment Left已经被R3 更新为0，R4会根据策略执行End.DX4操作，去掉IPv6外层报头，变成了一个普通的IPv6包，转发到指定的b主机。下图为R4从下面接口发出的包，同时也是主机b接收到的包。
