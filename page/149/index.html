避免使用vector bool

发表于 2021-01-12 分类于 c++ 阅读次数： Valine：

vector<bool>并不是一个stl容器

vector<bool>里面不是一个byte存储的，而是用一个bit来存储的

因为c++不能单独给一个bit去操作，因此我们使用[]访问时，获得的并不是true或false的引用，而是一个’proxy reference’

b是true或者false，因为这里进行了一个强制转换

而c的值就不是true或者false了

我们可以用deque<bool>来代替vector<bool>，deque<bool>中的true或false占用的是一个byte，里面保存的是真正的bool值

或者bitset，注意，bitset不是stl容器，而是c++标准库的一部分，大小在编译期固定，因此不支持插入和删除元素，不是迭代器，不支持iterator。压缩表示，每个值只占用一比特。如果不在意没有迭代器和动态改变大小，bitset正合适。
