1、C++内存分布，为什么栈比堆快

栈区

堆区

全局/静态区

文字常量区

程序代码区

1、堆中频繁调用malloc/free，会产生内存碎片，降低程序效率；栈由于其后进先出的特点，不会产生内存碎片

2、计算机底层对栈提供了一系列支持：分配专门的寄存器存储栈的地址，压栈和入栈由专门的指令。而堆是由c/c++函数库提供的，机制复杂，需要一系列分配内存，合并内存和释放内存的操作，效率较低

3、由于栈后进先出的特性，对栈数据的定位较快，而堆分配是随机的，定位较慢。

2、C++和C的区别

C++面向对象，C面向过程

最大的区别在于它们解决问题的思想不一样

C的主要特点是函数，是通过main函数来调用各个子函数。程序运行的顺序由程序员事先定好的

C++的主要特点是类，由类驱动程序运行。

C主要用于嵌入式领域，与硬件打交道

C++可以用于应用层开发、用户界面开发等领域

C++实际上由四部分组成：兼容C语言的部分 | 面向对象的部分 | 泛型编程，模板 | stl

3、TCP三次握手

1、客户端发送连接请求报文段，不携带应用层数据。SYN（同步位）= 1， seq = x（随机）

2、服务端为该TCP连接分配缓存和变量，并向客户端返回确认报文段，允许连接，不携带应用层数据

SYN = 1, ack = x + 1, seq = y;

3、客户端为该TCP分配缓存和变量，并向服务端返回确认的确认，可以携带数据

SYN = 0，ack = y + 1, seq = x + 1
