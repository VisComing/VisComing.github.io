课程概况与shell

发表于 2021-08-21 分类于 missing-semester 阅读次数： Valine：

众所周知，计算机擅长做重复的事情，但是我们经常忘记这一点也适用于我们使用计算机的方式，而不仅仅是利用计算机程序去帮我们求解问题。在从事与计算机相关的工作时，我们有很多触手可及的工具可以帮助我们更高效的解决问题。

bash

$表示我不是root用户，#表示我是root用户

echo

echo hello

用空格分隔参数

echo hello\ world

$PATH

指明了程序搜索的路径，$PATH用：分隔

可以使用which搜索可执行文件的位置

大多数命令接收标记(flag)和选项(option，需要跟值)

不需要值的称为标记，需要值的称为选项

权限

第一个字母：d表示目录，-表示普通文件

第一组：own权限

第二组：组权限

第三组：其他人的权限，既不是owner也不是group

对文件：读写执行，好理解

对目录：读：可以看目录里面有什么文件

​ 写：可以remove，create，重命名目录里面的文件。

如果你有文件的写权限，但是没有文件所在目录的写权限，那你可以修改文件的内容，但是无法删除文件，无法重命名文件。

目录的执行权限：是否可以进入该目录

在程序间创建连接

< file

> file

分别将程序的输入和输出重定向到文件

>> 追加

使用管道（ pipes ），我们能够更好的利用文件重定向。 | 操作符允许我们将一个程序的输出和另外一个程序的输入连接起来

sudo echo 3 > brightness

得到了错误

关于 shell，有件事我们必须要知道。|、>、和 < 是通过 shell 执行的，而不是被各个程序单独执行。 echo 等程序并不知道 | 的存在，它们只知道从自己的输入输出流中进行读写。 对于上面这种情况， shell (权限为您的当前用户) 在设置 sudo echo 前尝试打开 brightness 文件并写入，但是系统拒绝了 shell 的操作因为此时 shell 不是根用户。
